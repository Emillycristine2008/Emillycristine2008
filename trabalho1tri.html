Requisitos Funcionais Requisitos Não Funcionais Requisitos Inversos

Requisitos Funcionais (RF)

  • O sistema deve possibilitar que os alunos consultem os livros disponíveis.
  • O sistema deve permitir a realização de reservas de livros antecipadamente.
  • O sistema deve emitir notificações automáticas para lembrar a data de devolução.
  • O sistema deve sugerir livros específicos para cada turma.
  • O sistema deve registrar indicações de novos títulos.
  • O sistema deve disponibilizar acesso a livros em formato digital.
  • O sistema deve oferecer um espaço para que o professor recomende livros com base no perfil do aluno.
  • O sistema deve enviar avisos para prevenir esquecimentos.

Requisitos Não Funcionais (RNF)

  • O sistema precisa ser compatível com dispositivos móveis.
  • Deve reduzir o esforço manual e evitar deslocamentos desnecessários.
  • O acesso deve ser restrito a alunos e professores da escola, com controle de login.
  • A interface deve ser acessível, intuitiva e estimular a leitura.
  • Deve permitir consulta remota ao acervo, possibilitando acesso de qualquer lugar.
  • O controle de empréstimos e devoluções deve ser preciso, evitando extravios.
  • O acesso digital deve eliminar filas de espera por livros físicos.
  • As buscas por livros devem ser respondidas em até 2 segundos, mesmo com até 200 usuários simultâneos.
  • Deve exigir autenticação de usuários e garantir a proteção de dados sensíveis com criptografia.
  • O sistema deve operar 24 horas por dia, com no mínimo 99% de disponibilidade mensal e backups automáticos diários.
  • A interface deve ser intuitiva, permitindo que o usuário realize operações básicas (busca, empréstimo e devolução) em até 3 cliques, sem necessidade de treinamento.

Requisitos Inversos (RI)

  • O sistema não deve permitir que pessoas de fora da escola (comunidade externa à "Futuro do Saber") tenham acesso aos recursos ou ao acervo.
  • O sistema não deve permitir a reserva de um livro que já esteja emprestado ou reservado por outro aluno no mesmo período.
  • O sistema não deve permitir que ocorram atrasos sem o envio prévio de lembretes automáticos.
  • O sistema não deve permitir que o aluno precise se deslocar até a biblioteca para descobrir a indisponibilidade de um livro.
  • O sistema não deve permitir que o aluno perca o prazo de devolução por falta de notificação.
